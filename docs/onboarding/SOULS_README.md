# Souls (Agent Avatars)

AFO assumes each function (PM, UX, QA, Dev, etc.) can be operated by an agent avatar.

## What is a SOUL file?
A `SOUL.md` defines the persona, rules, outputs, and boundaries for an agent.

## How to start
1) Read `docs/onboarding/SETUP_QUESTIONS.md`
2) Create your own file under `souls/` (e.g., `souls/Alice.PM.SOUL.md`)
3) Use it as the shared state for your agent avatar

## Templates
- `souls/SOUL.template.md`
- `souls/PM.SOUL.template.md`
- `souls/UX.SOUL.template.md`
- `souls/QA.SOUL.template.md`
- `souls/DEV.SOUL.template.md`
