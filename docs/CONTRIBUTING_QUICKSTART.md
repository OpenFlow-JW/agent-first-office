# Contributing Quickstart (5–10 minutes)

Welcome! The fastest way to contribute is to improve docs, templates, or review lenses.

## 1) Pick a starting task
- See: `docs/GOOD_FIRST_ISSUES.md`
- Or open a small docs PR (typos, clarity, examples)

## 2) Fork & clone
1. Fork the repo on GitHub
2. Clone your fork:

```bash
git clone <your-fork-url>
cd agent-first-office
```

## 3) Create a branch
```bash
git checkout -b your-branch-name
```

## 4) Make changes
Good starter areas:
- `docs/` templates and explanations
- `docs/PRD.template.md`
- Add an example PRD (synthetic)

## 5) Commit with DCO sign-off
```bash
git add -A
git commit -s -m "docs: improve PRD template"
```

## 6) Open a PR
Push your branch and open a Pull Request.

## What NOT to contribute
- Customer-specific checklists / proprietary decision logic
- Private data, internal policies, secrets

If unsure, ask in an Issue first.
